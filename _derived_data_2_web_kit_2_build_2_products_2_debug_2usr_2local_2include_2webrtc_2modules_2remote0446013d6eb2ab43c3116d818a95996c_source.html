19 // A rate control implementation based on additive increases of
20 // bitrate when no over-use is detected and multiplicative decreases when
21 // over-uses are detected. When we think the available bandwidth has changes or

23 // multiplicatively.

55  // the current rate control state, the current target bitrate and the incoming
56  // bitrate. When in the "increase" state the bitrate will be increased either
57  // additively or multiplicatively depending on the rate control region. When

59  // incoming bitrate. When in the "hold" state the bitrate will be kept
