42 // The macros here print a message to stderr and abort under various
43 // conditions. All will accept additional stream messages. For example:
44 // RTC_DCHECK_EQ(foo, bar) << "I'm printed when foo != bar.";
45 //
46 // - RTC_CHECK(x) is an assertion that x is always true, and that if it isn't,
47 // it's better to terminate the process than to continue. During development,
48 // the reason that it's better to terminate might simply be that the error
49 // handling code isn't in place yet; in production, the reason might be that
50 // the author of the code truly believes that x will always be true, but that
51 // she recognizes that if she is wrong, abrupt and unpleasant process
52 // termination is still better than carrying on with the assumption violated.
53 //
54 // RTC_CHECK always evaluates its argument, so it's OK for x to have side
55 // effects.
56 //
57 // - RTC_DCHECK(x) is the same as RTC_CHECK(x)---an assertion that x is always
58 // true---except that x will only be evaluated in debug builds; in production
59 // builds, x is simply assumed to be true. This is useful if evaluating x is
60 // expensive and the expected cost of failing to detect the violated
61 // assumption is acceptable. You should not handle cases where a production
62 // build fails to spot a violated condition, even those that would result in
63 // crashes. If the code needs to cope with the error, make it cope, but don't
64 // call RTC_DCHECK; if the condition really can't occur, but you'd sleep
65 // better at night knowing that the process will suicide instead of carrying
66 // on in case you were wrong, use RTC_CHECK instead of RTC_DCHECK.
67 //
68 // RTC_DCHECK only evaluates its argument in debug builds, so if x has visible
69 // side effects, you need to write e.g.
70 // bool w = x; RTC_DCHECK(w);
71 //
72 // - RTC_CHECK_EQ, _NE, _GT, ..., and RTC_DCHECK_EQ, _NE, _GT, ... are
73 // specialized variants of RTC_CHECK and RTC_DCHECK that print prettier
74 // messages if the condition doesn't hold. Prefer them to raw RTC_CHECK and
75 // RTC_DCHECK.
76 //
77 // - RTC_FATAL() aborts unconditionally.
78 //
79 // TODO(ajm): Ideally, checks.h would be combined with logging.h, but
80 // consolidation with system_wrappers/logging.h should happen first.

89 // The actual stream used isn't important. We reference |ignored| in the code
90 // but don't evaluate it; this is to avoid "unused variable" warnings (we do so
91 // in a particularly convoluted way with an extra ?: because that appears to be
92 // the simplest construct that keeps Visual Studio from complaining about
93 // condition being unused).
94 #define RTC_EAT_STREAM_PARAMETERS(ignored) \
95  (true ? true : ((void)(ignored), true)) \
96  ? static_cast<void>(0) \
97  : rtc::FatalMessageVoidify() & rtc::FatalMessage("", 0).stream()

105 // RTC_CHECK dies with a fatal error if condition is not true. It is *not*
106 // controlled by NDEBUG or anything else, so the check will be executed

126 // Build the error message string. This is separate from the "Impl"
127 // function template because it is not performance critical and so can
128 // be out of line, while the "Impl" code should be inline. Caller
129 // takes ownership of the returned string.
130 template<class t1, class t2>
131 std::string* MakeCheckOpString(const t1& v1, const t2& v2, const char* names) {
132  std::ostringstream ss;
133  ss << names << " (" << v1 << " vs. " << v2 << ")";
134  std::string* msg = new std::string(ss.str());
135  return msg;
136 }
