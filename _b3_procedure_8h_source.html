64 // This represents B3's view of a piece of code. Note that this object must exist in a 1:1
65 // relationship with Air::Code. B3::Procedure and Air::Code are just different facades of the B3
66 // compiler's knowledge about a piece of code. Some kinds of state aren't perfect fits for either
67 // Procedure or Code, and are placed in one or the other based on convenience. Procedure always
68 // allocates a Code, and a Code cannot be allocated without an owning Procedure and they always
69 // have references to each other.
