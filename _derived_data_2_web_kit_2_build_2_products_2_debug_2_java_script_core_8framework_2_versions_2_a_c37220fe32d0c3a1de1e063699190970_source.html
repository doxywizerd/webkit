37 // BytecodeRewriter offers the ability to insert and remove the bytecodes including jump operations.
38 //
39 // We use the original bytecode offsets as labels. When you emit some jumps, you can specify the jump target by
40 // using the original bytecode offsets. These bytecode offsets are later converted appropriate values by the
41 // rewriter. And we also use the labels to represents the position the new bytecodes inserted.
42 //
43 // | [bytecode] | [bytecode] |
44 // offsets A B C
45 //
46 // We can use the above "A", "B", and "C" offsets as labels. And the rewriter has the ability to insert bytecode fragments
47 // before and after the label. For example, if you insert the fragment after "B", the layout becomes like this.
48 //
49 // | [bytecode] | [fragment] [bytecode] |
50 // offsets A B C
51 //
52 // And even if you remove some original bytecodes, the offset remains as labels. For example, when you remove the A's bytecode,
53 // the layout becomes like this.
54 //
55 // | | [bytecode] |
56 // offsets A B C
57 //
58 // And still you can insert fragments before and after "A".
59 //
60 // | [fragment] | [bytecode] |
61 // offsets A B C
62 //
63 // We can insert bytecode fragments "Before" and "After" the labels. This inserted position, either "Before" and "After",
64 // has effect when the label is involved with jumps. For example, when you have jump to the position "B",
65 //
66 // | [bytecode] | [bytecode] |
67 // offsets A B C
68 // ^
69 // jump to here.
70 //
71 // and you insert the bytecode before/after "B",
72 //
73 // | [bytecode] [before] | [after] [bytecode] |
74 // offsets A B C
75 // ^
76 // jump to here.
77 //
78 // as you can see, the execution jumping into "B" does not execute [before] code.
