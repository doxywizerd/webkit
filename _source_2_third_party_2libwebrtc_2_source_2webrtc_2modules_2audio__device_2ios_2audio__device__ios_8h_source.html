29 // Implements full duplex 16-bit mono PCM audio support for iOS using a
30 // Voice-Processing (VP) I/O audio unit in Core Audio. The VP I/O audio unit
31 // supports audio echo cancellation. It also adds automatic gain control,
32 // adjustment of voice-processing quality and muting.
33 //
34 // An instance must be created and destroyed on one and the same thread.
35 // All supported public methods must also be called on the same thread.
36 // A thread checker will RTC_DCHECK if any supported method is called on an
37 // invalid thread.
38 //
39 // Recorded audio will be delivered on a real-time internal I/O thread in the
40 // audio unit. The audio unit will also ask for audio data to play out on this
41 // same thread.

232  // Contains audio parameters (sample rate, #channels, buffer size etc.) for

236  // were granted by the OS. At this stage it is also possible to add a third

246  // FineAudioBuffer takes an AudioDeviceBuffer which delivers audio data
247  // in chunks of 10ms. It then allows for this data to be pulled in
248  // a finer or coarser granularity. I.e. interacting with this class instead
249  // of directly with the AudioDeviceBuffer one can ask for any number of
250  // audio data samples. Is also supports a similar scheme for the recording
251  // side.
252  // Example: native buffer size can be 128 audio frames at 16kHz sample rate.
253  // WebRTC will provide 480 audio frames per 10ms but iOS asks for 128
254  // in each callback (one every 8ms). This class can then ask for 128 and the
255  // FineAudioBuffer will ask WebRTC for new data only when needed and also
256  // cache non-utilized audio between callbacks. On the recording side, iOS
257  // can provide audio data frames of size 128 and these are accumulated until
258  // enough data to supply one 10ms call exists. This 10ms chunk is then sent
259  // to WebRTC and the remaining part is stored.
260  std::unique_ptr<FineAudioBuffer> fine_audio_buffer_;
261 
262  // Extra audio buffer to be used by the playout side for rendering audio.
263  // The buffer size is given by FineAudioBuffer::RequiredBufferSizeBytes().
264  std::unique_ptr<int8_t[]> playout_audio_buffer_;
