34 // Allocate one of these if you'd like to infer a constant value. Writes to the value should use
35 // notifyWrite(). So long as exactly one value had ever been written and invalidate() has never been
36 // called, and you register a watchpoint, you can rely on the inferredValue() being the one true
37 // value.
38 //
39 // Commonly used for inferring singletons - in that case each allocation does notifyWrite(). But you
40 // can use it for other things as well.

57  // For the purpose of deciding whether or not to watch this variable, you only need
58  // to inspect inferredValue(). If this returns something other than the empty
59  // value, then it means that at all future safepoints, this watchpoint set will be
60  // in one of these states:
61  //
62  // IsWatched: in this case, the variable's value must still be the
63  // inferredValue.
64  //
65  // IsInvalidated: in this case the variable's value may be anything but you'll
66  // either notice that it's invalidated and not install the watchpoint, or
67  // you will have been notified that the watchpoint was fired.
68  JSValue inferredValue() { return m_value.get(); }
