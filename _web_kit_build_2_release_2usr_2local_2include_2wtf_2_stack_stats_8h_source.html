34 // Define this flag to enable Stack stats collection. This feature is useful
35 // for getting a sample of native stack usage sizes.
36 //
37 // Enabling this will cause stats to be collected and written to a log file at
38 // various instrumented points in the code. It will result in noticeable
39 // performance loss. Hence, this should only be enable when you want to do
40 // some stats location in your local build. This code is provided here as a
41 // convenience for collecting that data. It is not meant to be enabled by
42 // default on release or debug builds.
