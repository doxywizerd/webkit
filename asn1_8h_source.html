350 /* The following macros and typedefs allow an ASN1_ITEM
351  * to be embedded in a structure and referenced. Since
352  * the ASN1_ITEM pointers need to be globally accessible
353  * (possibly from shared libraries) they may exist in
354  * different forms. On platforms that support it the
355  * ASN1_ITEM structure itself will be globally exported.
356  * Other platforms will export a function that returns
357  * an ASN1_ITEM pointer.
358  *
359  * To handle both cases transparently the macros below
360  * should be used instead of hard coding an ASN1_ITEM
361  * pointer in a structure.
362  *
363  * The structure will look like this:
364  *
365  * typedef struct SOMETHING_st {
366  * ...
367  * ASN1_ITEM_EXP *iptr;
368  * ...
369  * } SOMETHING;
370  *
371  * It would be initialised as e.g.:
372  *
373  * SOMETHING somevar = {...,ASN1_ITEM_ref(X509),...};
374  *
375  * and the actual pointer extracted with:
376  *
377  * const ASN1_ITEM *it = ASN1_ITEM_ptr(somevar.iptr);
378  *
379  * Finally an ASN1_ITEM pointer can be extracted from an
380  * appropriate reference with: ASN1_ITEM_rptr(X509). This
381  * would be used when a function takes an ASN1_ITEM * argument.
382  *
383  */
384
