46 // FramebufferAttachment implements a GL framebuffer attachment.
47 // Attachments are "light" containers, which store pointers to ref-counted GL objects.
48 // We support GL texture (2D/3D/Cube/2D array) and renderbuffer object attachments.
49 // Note: Our old naming scheme used the term "Renderbuffer" for both GL renderbuffers and for

67  // A framebuffer attachment points to one of three types of resources: Renderbuffers,
68  // Textures and egl::Surface. The "Target" struct indicates which part of the
69  // object an attachment references. For the three types:
70  // - a Renderbuffer has a unique renderable target, and needs no target index
71  // - a Texture has targets for every image and uses an ImageIndex
72  // - a Surface has targets for Color and Depth/Stencil, and uses the attachment binding
