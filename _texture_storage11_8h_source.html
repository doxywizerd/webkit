185  // These are members related to the zero max-LOD workaround.
186  // D3D11 Feature Level 9_3 can't disable mipmaps on a mipmapped texture (i.e. solely sample from level zero).
187  // These members are used to work around this limitation.
188  // Usually only mTexture XOR mLevelZeroTexture will exist.
189  // For example, if an app creates a texture with only one level, then 9_3 will only create mLevelZeroTexture.
190  // However, in some scenarios, both textures have to be created. This incurs additional memory overhead.
191  // One example of this is an application that creates a texture, calls glGenerateMipmap, and then disables mipmaps on the texture.
192  // A more likely example is an app that creates an empty texture, renders to it, and then calls glGenerateMipmap
193  // TODO: In this rendering scenario, release the mLevelZeroTexture after mTexture has been created to save memory.
194  ID3D11Texture2D *mLevelZeroTexture;
195  RenderTarget11 *mLevelZeroRenderTarget;
196  bool mUseLevelZeroTexture;
197
