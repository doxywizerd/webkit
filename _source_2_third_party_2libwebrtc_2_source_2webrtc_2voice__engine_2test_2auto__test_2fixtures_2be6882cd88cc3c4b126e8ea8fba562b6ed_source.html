30 // This convenient fixture sets up all voice engine interfaces automatically for
31 // use by testing subclasses. It allocates each interface and releases it once
32 // which means that if a tests allocates additional interfaces from the voice
33 // engine and forgets to release it, this test will fail in the destructor.
34 // It will not call any init methods.
35 //
36 // Implementation note:
37 // The interface fetching is done in the constructor and not SetUp() since
38 // this relieves our subclasses from calling SetUp in the superclass if they
39 // choose to override SetUp() themselves. This is fine as googletest will
40 // construct new test objects for each method.
