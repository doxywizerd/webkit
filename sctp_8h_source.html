160 /*
161  * Blocking I/O is enabled on any TCP type socket by default. For the UDP
162  * model if this is turned on then the socket buffer is shared for send
163  * resources amongst all associations. The default for the UDP model is that
164  * is SS_NBIO is set. Which means all associations have a separate send
165  * limit BUT they will NOT ever BLOCK instead you will get an error back
166  * EAGAIN if you try to send too much. If you want the blocking semantics you
167  * set this option at the cost of sharing one socket send buffer size amongst
168  * all associations. Peeled off sockets turn this option off and block. But
169  * since both TCP and peeled off sockets have only one assoc per socket this
170  * is fine. It probably does NOT make sense to set this on SS_NBIO on a TCP
171  * model OR peeled off UDP model, but we do allow you to do so. You just use
172  * the normal syscall to toggle SS_NBIO the way you want.
173  *
174  * Blocking I/O is controlled by the SS_NBIO flag on the socket state so_state
175  * field.
176  */

220 /* VRF (virtual router feature) and multi-VRF support
221  * options. VRF's provide splits within a router
222  * that give the views of multiple routers. A
223  * standard host, without VRF support, is just
224  * a single VRF. If VRF's are supported then
225  * the transport must be VRF aware. This means
226  * that every socket call coming in must be directed
227  * within the endpoint to one of the VRF's it belongs
228  * to. The endpoint, before binding, may select
229  * the "default" VRF it is in by using a set socket
230  * option with SCTP_VRF_ID. This will also
231  * get propagated to the default VRF. Once the
232  * endpoint binds an address then it CANNOT add
233  * additional VRF's to become a Multi-VRF endpoint.
234  *
235  * Before BINDING additional VRF's can be added with
236  * the SCTP_ADD_VRF_ID call or deleted with
237  * SCTP_DEL_VRF_ID.
238  *
239  * Associations are ALWAYS contained inside a single
240  * VRF. They cannot reside in two (or more) VRF's. Incoming
241  * packets, assuming the router is VRF aware, can always
242  * tell us what VRF they arrived on. A host not supporting
243  * any VRF's will find that the packets always arrived on the
244  * single VRF that the host has.
245  *
246  */
247
