34 // This class's only job is to hold onto the list of ScopeOffsets for each argument that a
35 // function has. Most of the time, the BytecodeGenerator will create one of these and it will
36 // never be modified subsequently. There is a rare case where a ScopedArguments object is created
37 // and aliases one of these and then decides to modify it; in that case we do copy-on-write. This
38 // makes sense because such modifications are so uncommon. You'd have to do something crazy like
39 // "delete arguments[i]" or some variant of defineOwnProperty.
