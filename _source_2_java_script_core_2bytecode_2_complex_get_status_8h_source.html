37 // This class is useful for figuring out how to inline a cached get-like access. We
38 // say "get-like" because this is appropriate for loading the GetterSetter object in
39 // a put_by_id that hits a setter. Notably, this doesn't figure out how to call
40 // accessors, or even whether they should be called. What it gives us, is a way of
41 // determining how to load the value from the requested property (identified by a
42 // StringImpl* uid) from an object of the given structure in the given CodeBlock,
43 // assuming that such an access had already been cached by Repatch (and so Repatch had
44 // already done a bunch of safety checks). This doesn't reexecute any checks that
45 // Repatch would have executed, and for prototype chain accesses, it doesn't ask the
46 // objects in the prototype chain whether their getOwnPropertySlot would attempt to
47 // intercept the access - so this really is only appropriate if you already know that
48 // one of the JITOperations had OK'd this for caching and that Repatch concurred.
49 //
50 // The typical use pattern is something like:
51 //
52 // ComplexGetStatus status = ComplexGetStatus::computeFor(...);
53 // switch (status.kind()) {
54 // case ComplexGetStatus::ShouldSkip:
55 // // Handle the case where this kind of access is possibly safe but wouldn't
56 // // pass the required safety checks. For example, if an IC gives us a list of
57 // // accesses and one of them is ShouldSkip, then we should pretend as if it
58 // // wasn't even there.
59 // break;
60 // case ComplexGetStatus::TakesSlowPath:
61 // // This kind of access is not safe to inline. Bail out of any attempst to
62 // // inline.
63 // break;
64 // case ComplexGetStatus::Inlineable:
65 // // The good stuff goes here. If it's Inlineable then the other properties of
66 // // the 'status' object will tell you everything you need to know about how
67 // // to execute the get-like operation.
68 // break;
69 // }
