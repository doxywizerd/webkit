39 // Often, we create threads that have this as their body:
40 //
41 // for (;;) {
42 // {
43 // LockHolder locker(m_lock);
44 // for (;;) {
45 // [1] stuff that could break, return, or fall through;
46 // m_condition.wait(m_lock);
47 // }
48 // }
49 //
50 // [2] do work;
51 // }
52 //
53 // When we do this, we don't always do a good job of managing this thread's lifetime, which may lead
54 // to this thread sitting around even when it is not needed.
55 //
56 // AutomaticThread is here to help you in these situations. It encapsulates a lock, a condition
57 // variable, and a thread. It will automatically shut the thread down after 1 second of inactivity.
58 // You use AutomaticThread by subclassing it, and put any state that is needed between [1] and [2]
59 // in the subclass.
60 //
61 // The terminology we use is:
62 //
63 // [1] PollResult AutomaticThread::poll()
64 // [2] WordResult AutomaticThread::work()
65 //
66 // Note that poll() and work() may not be called on the same thread every time, since this will shut
67 // down the thread as necessary. This is legal since m_condition.wait(m_lock) can drop the lock, and
68 // so there is no reason to keep the thread around.

103  // Note that if you drop all of your references to an AutomaticThread then as soon as there is a
104  // second during which it doesn't get woken up, it will simply die on its own. This is a
105  // permanent kind of death where the AutomaticThread object goes away, rather than the temporary
106  // kind of death where AutomaticThread lives but its underlying thread dies. All you have to do
107  // to prevent permanent death is keep a ref to AutomaticThread. At time of writing, every user of
108  // AutomaticThread keeps a ref to it and does join() as part of the shutdown process, so only the
109  // temporary kind of automatic death happens in practice. We keep the permanent death feature
110  // because it leads to an easy-to-understand reference counting discipline (AutomaticThread holds
111  // strong ref to AutomaticThreadCondition and the underlying thread holds a strong ref to
112  // AutomaticThread).
