33 // This is an Arguments-class object that we create when you do function.arguments, or you say
34 // "arguments" inside a function in strict mode. It behaves almpst entirely like an ordinary
35 // JavaScript object. All of the arguments values are simply copied from the stack (possibly via
36 // some sophisticated ValueRecovery's if an optimizing compiler is in play) and the appropriate
37 // properties of the object are populated. The only reason why we need a special class is to make
38 // the object claim to be "Arguments" from a toString standpoint, and to avoid materializing the
39 // caller/callee/@@iterator properties unless someone asks for them.
