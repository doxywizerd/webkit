78 // The Widget class serves as a base class for three kinds of objects:
79 // (1) Scrollable areas (ScrollView)
80 // (2) Scrollbars (Scrollbar)
81 // (3) Plugins (PluginView)
82 //
83 // A widget may or may not be backed by a platform-specific object (e.g., HWND on Windows, NSView on Mac, QWidget on Qt).
84 //
85 // Widgets are connected in a hierarchy, with the restriction that plugins and scrollbars are always leaves of the
86 // tree. Only ScrollViews can have children (and therefore the Widget class has no concept of children).
87 //
88 // The rules right now for which widgets get platform-specific objects are as follows:
89 // ScrollView - Mac
90 // Scrollbar - Mac, Gtk
91 // Plugin - Mac, Windows (windowed only), Qt (windowed only, widget is an HWND on windows), Gtk (windowed only)
